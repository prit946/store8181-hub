How to Use Z Report
Submitting a Z Report

Tap a date on the calendar — green = done, red = missing. Fill in Sales, Tax, CC, Cash from the Z tape at closing. Pick the employee name and tap Submit.

Correcting an Entry

Tap any green (already submitted) date. The form pre-fills with existing data. Edit and submit to overwrite.

Missing Entries Banner

Red chips below the calendar show dates with no entry in the last 60 days. Tap any chip to jump to that date.

How Your Data Works

All Z report data is saved directly on this device (browser local storage). No account, no internet, no server — it just works. This is the same technology that saves your phone's app preferences and it is very reliable for store operations.

One Device Per Store (Recommended)

The cleanest setup is one designated device — a store tablet or the register computer — that everyone uses to submit Z reports. All history stays in one place, nothing gets out of sync.

Employee Join Link

The Employee Join Link in Settings configures a second device automatically. The employee just clicks the link and enters their PIN — the store name is pre-filled. Useful for a backup device, but note that each device stores its own reports independently.

Back Up Your Data (Important)

Browser storage can be cleared if someone taps "Clear Site Data" or reinstalls the browser. Export a JSON backup from Settings → Data Backup every month or whenever you have a lot of entries you can't afford to lose. Save it to Google Drive, email it to yourself, or paste the CSV into a spreadsheet.

My Stores — Multi-Store

Store 8181 uses Google Sheets (existing setup). Additional stores you own use Firebase — one database holds all of them. Set the Firebase URL once, then add each store below.

Firebase Connection
One-time Firebase setup (free, 5 min) 1. console.firebase.google.com → Create project → Continue
2. Realtime Database → Create Database → Start in test mode → Enable
3. Copy the URL (looks like https://your-project-default-rtdb.firebaseio.com)
4. Rules → set {"rules"{".read":true,".write":true}}
